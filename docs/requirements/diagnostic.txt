PAGE PURPOSE
The Strategic Diagnostic page is the formal entry point into the Axis ecosystem.
This page should not feel like a contact form, inquiry page, or lead capture funnel.
It should feel like a private evaluation gateway.
The purpose is to determine whether Axis can create leverage inside the organization and where the system should be applied first.
The visitor should understand:
Axis does not begin with a sales call. Axis begins with diagnosis.

SECTION 06
The Intake Experience
Heading
Built as a Website Intake, Not a Native Form
Copy
The Axis Alignment Application should be built directly inside the Axis website so the experience can be fully controlled across design, spacing, dropdown behavior, mobile responsiveness, and brand presentation.
The frontend must feel custom, premium, and structured.
Required Frontend Features
Multi-step application
Progress indicator
Clean dropdown behavior
Other/type option on relevant questions
Mobile-first layout
Save and continue if possible
Clear validation
No fields hidden by browser bars
Fully branded Axis interface
Secure and confidential submission experience
Backend Requirements
Form submission handling
AI prompt processing
Scoring logic
Email automation
Internal storage
ClickUp or Airtable routing
Team notification
Branded email or PDF output if needed
Important Direction
Do not use the native ClickUp form UI for the public experience.
ClickUp can still be used behind the scenes for routing, storage, and internal task management.
